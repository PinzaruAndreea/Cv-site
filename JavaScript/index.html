My first Project- a Quiz

I loved doing this project. Firstly, I had to do all the CSS work. The course instructor used bootstrap, but, since I had just taken the CSS course, I wanted to practice my newly aquired skills. I tried to stay as authentic to the piece as possible.

In my JS file, as explained in the CV on the homepage, I defined a few constants and one local -let- variable. I added an even listener on the submit button that calculated how many of the answers chosen by the user match the constant correctAnswers. I used a forEach() to go through all of them and add them to the let scores which will update with the score.

I added a retry button for the script to refresh and be able to do the quiz again.

Second project: To do List

This one was a little bit more complicated, to say the least. At the beginning, I did the CSS work with the same idea in mind as on the previous project, the issue with the functionality of this one was that when I would refresh the page, everything that I previously input, would delete itself. This got me thinking, I realized that this was not actually linked to some sort of database to store everything that I add there. In response to that, I had implemented firestore (Google Firebase) in my project by initializing the firebaseConfig variable and storing the firestore in the db constant. Next, I stored an arrow function in the addItem constant that will add a new li item into the list after you submit it in the add a new to do box (I later added a submit event for the change to take place on the page) Then comes the deleteItem constant that checks to delete the item with the same attribute ("data-id") as the one you press.(Again, an event listener to check if the delete button is pressed. There's also a filter arrow function and search event listener ("keyup") that filters out all the combination of letters that does not match what it is typed in there and displays only what matches. Lastly, there's an option to unsubscribe onSnapshot that listens for the changes and, if you press the button, you will no longer store anything in the firebase.

Third project: Weather app

For this project I had to create two JS files to keep the script as clean as possible. One was a reference to the accuWeather API and the other was for the entire functionality of the application. In the forecast.js I created a class with a constructor referencing the key, the weather URI and the city URI. After that I created three asynchronous functions to retrieve all the weather data from a city that will be later submitted by the user.

The app.js file contains arrow functions that will return the requested weather information once the promise has been fulfilled (so the data was fetched). For styling purposes, I also added a few svg files, out of which only those that match the weather conditions and time (day/night) will be displayed.

Fourth project: Live Chat

In this one I split the script in three files for a cleaner outlook: sandbox.js, ui.js, app.js. In the sandbox file I defined a class called Chatroom with a constructor that specifies that the room and username are the same as the one chosen by the user. I also retrieved the chats from the firebase. Then I wrote an asynchronous function with three constants: one to display the time the message was sent, one for the chats (that is created under a specific criteria -message, username, room and timestamp) and another response constant that waits for a promise, in this case, it is for all future chats that will be input. The getChats() looks for messages from the database for the chosen room and retrieves the requested data. There are also updateName and updateRoom functions that change the username and the room. For ui.js I created another class ChatUI that actually adds all the data on the page. The last file is for functionality, I created a few event listeners so that the previous functions can be called back and put to good use. :)

Fifth project: UI library

This was a more simple project to implement in JS. The complicated part came with the addition of webpack and babel to the project. In a nutshell, they take all the js files that are in a directory and bundles them in a more "primitive" JS outlook to make the files responsive to different web browsers. I created 4 js files to define the cool tools that were made in this project. Then I initialized them all in the index.js file. The script mentioned in the index.html file, however, is the bundle.js file (which contains all the bundled js code made through the webpack boilerplate, which was both webpack and babel.

As far as what these items do, I first made a tooltip, which displays a message once you hover over the item. The dropdown is, I believe pretty self explanatory, once you click on the "\/" an active class is added to the content and the information is displayed and, if you click again, the class is removed. For the tabs I did about the same thing, but I asked the script to look for the attribute ("data-target") that the user clicks on and display the content. And lastly, for the snackbar, I used a setTimeout arrow function that will display the bar for 5000ms and then remove it from the body.
